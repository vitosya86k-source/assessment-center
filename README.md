# NeuroHRV Dashboard — Методология и формат

## 📚 Методологическая база

### Основные источники

1. **Task Force of ESC/NASPE (1996)** — "Heart rate variability: Standards of measurement, physiological interpretation, and clinical use"
   - Золотой стандарт интерпретации HRV
   - Определяет временные и частотные показатели
   - Устанавливает нормы для 5-минутных записей

2. **Баевский Р.М. (1984, 2001)** — "Анализ вариабельности сердечного ритма при использовании различных электрокардиографических систем"
   - Индекс напряжения (Stress Index / SI)
   - Оценка функциональных резервов организма
   - Широко используется в СНГ и космической медицине

3. **Kubios HRV Scientific** — современные алгоритмы
   - Readiness Score
   - Biological Age estimation
   - Комплексные интегральные показатели

## 🎯 7 осей паутинковой диаграммы

### Принцип отбора
Выбраны показатели, которые:
- Имеют клиническую валидацию
- Рассчитываются из доступных данных (HR + расчётные RR)
- Релевантны для нейроассессмента (стресс, адаптация, ресурсы)
- Понятны при интерпретации

---

### RD — Readiness (Готовность)

**Полное название:** Комплексная готовность к нагрузкам

**Источник:** Kubios HRV, адаптация Task Force

**Что измеряет:** Интегральный показатель способности нервной системы адаптироваться к физическим и умственным нагрузкам

**Формула расчёта:**
```
RD = 0.3×RMSSD_norm + 0.25×SDNN_norm + 0.2×(100-SI_norm) + 0.15×HF_norm + 0.1×LF_HF_balance
```
где _norm — нормализация к шкале 0-100

**Диапазоны:**
| Значение | Уровень | Интерпретация для нейроассессмента |
|----------|---------|-----------------------------------|
| 80-100% | Отличный | Готов к сложным задачам и лидерству |
| 60-79% | Хороший | Стандартные задачи, может быть ментором |
| 40-59% | Норма | Рутинные задачи, избегать стресса |
| <40% | Низкий | Простые задачи, требуется поддержка |

**Возрастная коррекция:** После 50 лет нормальными считаются значения на 10-15% ниже

---

### SR — Stress Resistance (Стрессоустойчивость)

**Полное название:** Устойчивость к стрессовым воздействиям

**Источник:** Баевский Р.М. — Stress Index (инверсия)

**Что измеряет:** Способность сохранять функциональное состояние при стрессе. Основан на активности парасимпатической НС.

**Формула расчёта:**
```
SI (Stress Index) = AMo / (2 × Mo × MxDMn)
где:
  AMo — амплитуда моды (% интервалов в диапазоне моды)
  Mo — мода RR интервалов
  MxDMn — вариационный размах

SR% = 100 - min(SI / 6, 100)
```

**Диапазоны Stress Index (по Баевскому):**
| SI | Состояние | SR% |
|----|-----------|-----|
| <50 | Расслабление | 92-100% |
| 50-150 | Норма (рабочий тонус) | 75-92% |
| 150-300 | Напряжение | 50-75% |
| 300-500 | Выраженное напряжение | 17-50% |
| >500 | Перенапряжение | <17% |

**Критический порог:** SI > 600 — высокий риск срыва адаптации

---

### AD — Adaptability (Адаптивность)

**Полное название:** Общая адаптационная способность

**Источник:** Task Force — SDNN как маркер общей вариабельности

**Что измеряет:** Способность сердечно-сосудистой системы адаптироваться к изменяющимся условиям. Включает все частотные компоненты.

**Формула расчёта:**
```
SDNN = √(Σ(RRᵢ - mean_RR)² / (n-1))

AD% = min(SDNN / 100 × 100, 100)
Нормализация: 0 мс = 0%, 100+ мс = 100%
```

**Диапазоны SDNN (5-минутная запись, Task Force):**
| SDNN | Уровень | AD% |
|------|---------|-----|
| >100 мс | Отличный | 100% |
| 50-100 мс | Хороший | 50-100% |
| 30-50 мс | Сниженный | 30-50% |
| <30 мс | Критический | <30% |

**Клиническое значение:** SDNN <50 мс связан с 5.3-кратным увеличением риска неблагоприятных кардиособытий (Kleiger, 1987)

---

### FL — Flexibility (Гибкость НС)

**Полное название:** Гибкость нервной системы

**Источник:** Task Force — pNN50; Poincaré — SD1

**Что измеряет:** Способность к быстрой адаптации и переключению между задачами. Специфический маркер вагусной активности.

**Формула расчёта:**
```
pNN50 = (количество NN интервалов с разницей >50мс) / (общее количество NN) × 100%
SD1 = √(0.5 × RMSSD²)

FL% = 0.6 × pNN50_norm + 0.4 × SD1_norm
```

**Диапазоны:**
| pNN50 | SD1 | Уровень | FL% |
|-------|-----|---------|-----|
| >15% | >40 мс | Отличный | 80-100% |
| 5-15% | 15-40 мс | Норма | 40-80% |
| 1-5% | 5-15 мс | Сниженный | 15-40% |
| <1% | <5 мс | Критический | <15% |

**Возрастная особенность:** Резкое снижение после 40 лет. У пожилых норма pNN50 >3%

**Медицинский маркер:** pNN50 <1% при диабете указывает на автономную нейропатию

---

### RC — Recovery (Восстановление)

**Полное название:** Способность к восстановлению

**Источник:** Task Force — HF power, RMSSD

**Что измеряет:** Активность парасимпатической НС, отвечающей за восстановление и расслабление между задачами.

**Формула расчёта:**
```
RMSSD = √(Σ(RRᵢ₊₁ - RRᵢ)² / (n-1))
HF power = мощность спектра 0.15-0.4 Гц

RC% = 0.5 × RMSSD_norm + 0.5 × HF_norm
```

**Диапазоны RMSSD:**
| RMSSD | HF power | Уровень | RC% |
|-------|----------|---------|-----|
| >50 мс | >500 ms² | Отличный | 80-100% |
| 25-50 мс | 200-500 ms² | Хороший | 50-80% |
| 15-25 мс | 100-200 ms² | Норма | 30-50% |
| <15 мс | <100 ms² | Сниженный | <30% |

**Возрастная коррекция:** RMSSD снижается на 1-2 мс каждые 10 лет. В 60+ нормой считается >15 мс.

---

### EN — Endurance (Выносливость)

**Полное название:** Устойчивость к длительным нагрузкам

**Источник:** Poincaré — SD2

**Что измеряет:** Долгосрочная вариабельность, способность поддерживать производительность в течение длительного времени.

**Формула расчёта:**
```
SD2 = √(2×SDNN² - 0.5×RMSSD²)

EN% = min(SD2 / 100 × 100, 100)
```

**Диапазоны SD2:**
| SD2 | Уровень | EN% | Интерпретация |
|-----|---------|-----|---------------|
| >70 мс | Высокий | 70-100% | Высокая выносливость |
| 50-70 мс | Хороший | 50-70% | Стабильная работоспособность |
| 30-50 мс | Норма | 30-50% | Средняя устойчивость |
| <30 мс | Низкий | <30% | Быстрое истощение ресурсов |

---

### BL — Balance (Вегетативный баланс)

**Полное название:** Симпато-вагальный баланс

**Источник:** Task Force — LF/HF ratio

**Что измеряет:** Соотношение активности симпатической и парасимпатической НС. Определяет оптимальность состояния для разных типов задач.

**Формула расчёта:**
```
LF/HF = LF_power / HF_power

BL% конвертируется нелинейно:
  LF/HF 0.5-2.0 → 70-100% (оптимальный баланс)
  LF/HF 2.0-4.0 → 40-70% (симпатическое преобладание)
  LF/HF <0.5 → 50-70% (парасимпатическое преобладание)
  LF/HF >4.0 → <40% (стресс)
```

**Диапазоны LF/HF (Task Force):**
| LF/HF | Состояние | BL% | Рекомендации |
|-------|-----------|-----|--------------|
| <0.5 | ПНС доминирует | 50-70% | Творческие задачи, брейншторминг |
| 0.5-2.0 | Баланс | 70-100% | Оптимально для большинства задач |
| 2.0-4.0 | СНС доминирует | 40-70% | Сложные вызовы, дедлайны |
| >4.0 | Стресс | <40% | Снижение эффективности |

**Критический порог:** LF/HF > 6.0 — высокий риск импульсивных решений

## 🔢 Алгоритм расчёта общего балла

```python
def calculate_overall_score(metrics):
    """
    Веса подобраны для контекста нейроассессмента:
    - Стрессоустойчивость и Готовность — приоритет
    - Баланс и Восстановление — важны для длительных сессий
    """
    weights = {
        'RD': 0.20,  # Готовность — ключевой
        'SR': 0.20,  # Стрессоустойчивость — ключевой
        'AD': 0.15,  # Адаптивность
        'FL': 0.10,  # Гибкость
        'RC': 0.15,  # Восстановление
        'EN': 0.10,  # Выносливость
        'BL': 0.10   # Баланс
    }
    
    score = sum(metrics[k] * weights[k] for k in weights)
    return round(score)
```

## 📐 Алгоритм определения событий (всплесков)

```python
def detect_events(hr_data, si_data, timestamps):
    """
    Определяет события — отклонения от скользящей средней
    """
    events = []
    window = 30  # секунд для скользящей средней
    threshold_hr = 0.15  # 15% отклонение
    threshold_si = 0.50  # 50% отклонение
    
    for i in range(window, len(hr_data)):
        mean_hr = np.mean(hr_data[i-window:i])
        mean_si = np.mean(si_data[i-window:i])
        
        hr_deviation = (hr_data[i] - mean_hr) / mean_hr
        si_deviation = (si_data[i] - mean_si) / mean_si if mean_si > 0 else 0
        
        if hr_deviation > threshold_hr or si_deviation > threshold_si:
            events.append({
                'start': timestamps[i],
                'hr_change': f"{mean_hr:.0f}→{hr_data[i]:.0f}",
                'hr_percent': f"+{hr_deviation*100:.0f}%",
                'si_change': f"{mean_si:.0f}→{si_data[i]:.0f}",
                'type': classify_event(hr_deviation, si_deviation)
            })
    
    # Объединяем последовательные события
    return merge_consecutive_events(events)

def classify_event(hr_dev, si_dev):
    if hr_dev > 0.20:
        return "Пиковая активация"
    elif hr_dev > 0.10:
        return "Активация симпатики"
    elif si_dev > 0.50:
        return "Стрессовая реакция"
    else:
        return "Умеренное напряжение"
```
