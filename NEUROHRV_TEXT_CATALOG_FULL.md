### 1.2 Правило выбора варианта текста

Для каждого уровня доступно **3 варианта** формулировки.  
Выбор: `hash(user_id + date + metric_code) % 3` — так один и тот же пользователь в один день видит одну формулировку, но при повторном замере в другой день — другую.

### 1.3 Правило выбора контекста

```
Если context == "assessment":  → 🧠 когнитивные формулировки
Если context == "sport":       → 💪 физические формулировки
Если context == "universal":   → комбинированная формулировка (по умолчанию)
```

## 10. SWOT — Правила формирования

### 10.1 Сильные стороны [+]

**Правило:** Ось попадает в «Сильные», если значение ≥ 60%

**Формат формулировки:** `"{Название} {value}% — {описание_уровня}"`

Описания берутся из соответствующего раздела оси, контекст — из `context`.

**Максимум в блоке:** 3–4 позиции (если больше — показываем топ-3 по значению)

### 10.2 Ограничения [-]

**Правило:** Ось попадает в «Ограничения», если значение < 40%

**Формат:** `"{Название} {value}% — {описание_уровня}"`

Для критических (<15%) добавлять пометку: `"⚠️ критически"` перед описанием.

**Максимум в блоке:** 3–4 позиции

### 10.3 Возможности [*]

**Правило:** Формируются из рекомендаций для осей в зонах «medium» и «low»

**Формат:** Конкретные действия, привязанные к слабым осям

**Каталог возможностей по осям:**

| Ось | Если `low`/`medium` | Возможность (вариант 1) | Возможность (вариант 2) | Возможность (вариант 3) |
|-----|---------------------|------------------------|------------------------|------------------------|
