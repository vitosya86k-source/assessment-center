## 2. ОШИБКА: Биологический возраст 74 года при HRV 31.4 мс

### Проблема:
При RMSSD ≈ 31 мс биологический возраст должен быть ~43-45 лет, а показывает 74 года.

### Текущая формула (видимо неправильно реализована):
```python
# Где-то ошибка — возможно rmssd передаётся в неправильных единицах
# или используется другая переменная
```

### Правильная формула:
```python
def calculate_biological_age(rmssd, sdnn, stress_index):
    """
    rmssd в миллисекундах (мс)
    """
    # Базовая оценка через RMSSD
    if rmssd >= 50:
        base_age = 20
    elif rmssd >= 40:
        base_age = 20 + (50 - rmssd) * 1.0  # 50→20, 40→30
    elif rmssd >= 30:
        base_age = 30 + (40 - rmssd) * 1.5  # 40→30, 30→45
    elif rmssd >= 20:
        base_age = 45 + (30 - rmssd) * 1.5  # 30→45, 20→60
    elif rmssd >= 10:
        base_age = 60 + (20 - rmssd) * 1.5  # 20→60, 10→75
    else:
        base_age = 75 + (10 - rmssd) * 2.0
    
    # Коррекция по SDNN
    if sdnn > 100:
        base_age -= 5
    elif sdnn < 30:
        base_age += 5
    
    # Коррекция по Stress Index
    if stress_index < 100:
        base_age -= 3
    elif stress_index > 300:
        base_age += 5
    
    return max(18, min(90, round(base_age)))
```

### Проверка:
При rmssd = 31.4 мс:
- base_age = 30 + (40 - 31.4) * 1.5 = 30 + 12.9 = 42.9
- sdnn ≈ 31, < 30? Нет, но близко → без коррекции или +5
- stress_index = 75.9 < 100 → -3
- Итого: ~40-45 лет (НЕ 74!)

### Что проверить в коде:
1. Убедиться что `rmssd` передаётся в мс, а не в других единицах
2. Проверить что не перепутаны переменные
3. Добавить отладочный вывод для проверки входных значений

## 3. ОШИБКА: BL (Баланс) = 6% при LF/HF = 10.7

### Проблема:
По старой формуле должно быть 0%, показывает 6%. Формула должна быть более плавной.

### Исправленная формула:
```python
def balance_score(lf_hf_ratio):
    """
    Более плавное снижение для высоких LF/HF
    """
    if 0.5 <= lf_hf_ratio <= 2.0:
        return 100  # Оптимальный баланс
    elif lf_hf_ratio < 0.5:
        return max(50, 50 + lf_hf_ratio * 100)  # 0→50%, 0.5→100%
    elif lf_hf_ratio <= 4.0:
        return max(40, 100 - (lf_hf_ratio - 2) * 30)  # 2→100%, 4→40%
    else:
        # ИСПРАВЛЕНО: коэффициент 5 вместо 10 для плавного снижения
        return max(5, 40 - (lf_hf_ratio - 4) * 5)  # 4→40%, 8→20%, 12→0%
```

### Проверка:
При LF/HF = 10.7:
- BL = max(5, 40 - (10.7 - 4) * 5) = max(5, 40 - 33.5) = max(5, 6.5) = 6.5% ≈ 7%

Это близко к 6%, значит формула уже исправлена, но минимум должен быть 5%, не 0%.

## 5. ОШИБКА: Динамика — нет списка событий (всплесков)

### Проблема:
На дашборде "Динамика" отсутствует блок с событиями. Должны отображаться:
- Всплески ВВЕРХ (📈 активация)
- Всплески ВНИЗ (📉 снижение)

### Что добавить:
```python
def detect_events(hr_data, si_data, timestamps, 
                  hr_threshold=0.12, si_threshold=0.30, window=30):
    """
    Определение событий — отклонений от скользящей средней
    """
    events = []
    
    for i in range(window, len(hr_data)):
        mean_hr = np.mean(hr_data[i-window:i])
        mean_si = np.mean(si_data[i-window:i])
        
        hr_dev = (hr_data[i] - mean_hr) / mean_hr if mean_hr > 0 else 0
        si_dev = (si_data[i] - mean_si) / mean_si if mean_si > 0 else 0
        
        # Всплеск ВВЕРХ
        if hr_dev > hr_threshold or si_dev > si_threshold:
            events.append({
                'index': i,
                'timestamp': timestamps[i],
                'type': 'up',  # 📈
                'hr_before': round(mean_hr),
                'hr_after': hr_data[i],
                'hr_change': f"+{hr_dev*100:.0f}%"
            })
        
        # Всплеск ВНИЗ
        elif hr_dev < -hr_threshold or si_dev < -si_threshold:
            events.append({
                'index': i,
                'timestamp': timestamps[i],
                'type': 'down',  # 📉
                'hr_before': round(mean_hr),
                'hr_after': hr_data[i],
                'hr_change': f"{hr_dev*100:.0f}%"
            })
    
    # Объединить последовательные события
    return merge_consecutive_events(events)
```

### Отображение на дашборде:
```
📍 СОБЫТИЯ

📈 15:48:30 — 15:48:45 (15 сек)
   HR: 58 → 65 (+12%) 
   Активация

📉 15:50:10 — 15:50:25 (15 сек)
   HR: 62 → 55 (-11%)
   Снижение активности
```

## 9. ОШИБКА: Противоречие SR 94% при плохих показателях

### Анализ:
- SR (Стрессоустойчивость) = 94% 
- Но LF/HF = 10.7 (критически высокий!)
- pNN50 = 0% (критически низкий!)
- BL = 6% (критически низкий!)

SR рассчитывается только из Stress Index (75.9 — норма), поэтому высокий.
Но это НЕ отражает реальную картину — организм явно в стрессе по другим показателям.

### Рекомендация:
Добавить в расчёт SR влияние LF/HF:
```python
def stress_resistance_score(stress_index, lf_hf_ratio):
    """
    Стрессоустойчивость с учётом LF/HF
    """
    # Базовый расчёт из SI
    if stress_index <= 50:
        sr_base = 100
    elif stress_index <= 150:
        sr_base = 100 - (stress_index - 50) * 0.25
    elif stress_index <= 300:
        sr_base = 75 - (stress_index - 150) * 0.33
    elif stress_index <= 500:
        sr_base = 25 - (stress_index - 300) * 0.10
    else:
        sr_base = max(0, 5 - (stress_index - 500) * 0.01)
    
    # Коррекция по LF/HF (если сильный дисбаланс — снижаем SR)
    if lf_hf_ratio > 4.0:
        penalty = min(30, (lf_hf_ratio - 4) * 5)  # до -30%
        sr_base = max(0, sr_base - penalty)
    
    return round(sr_base)
```

При SI=75.9, LF/HF=10.7:
- sr_base = 100 - (75.9 - 50) * 0.25 = 100 - 6.5 = 93.5%
- penalty = min(30, (10.7 - 4) * 5) = min(30, 33.5) = 30
- SR = 93.5 - 30 = 63.5%

Это более реалистично!
