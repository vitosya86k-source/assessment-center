### 1.1 PNG «Нейроассессмент» (Профиль) — ОСНОВНЫЕ ПРОБЛЕМЫ

**Что сейчас ломается:**

а) **Нижняя панель метрик обрезается и налезает.**  
Внизу дашборда идут два ряда показателей:
```
Ряд 1: Stress Index | HR | Mean RR | RMSSD | SDNN | LF/HF
Ряд 2: LF n.u. | HF n.u. | SD1/SD2 | VLF | Total | Дыхание
```
Эти два ряда сжаты, цветные индикаторы (🟢🟡🔴) налезают на текст. На скриншоте видно, что надписи типа "LF n.u.", "HF n.u." перекрывают значения.

**Как исправить:**
- Увеличить высоту фигуры (figure height). Сейчас, видимо, ~800px. Нужно минимум **900–950px** при 1200px ширины.
- Нижнюю панель метрик разместить с явным отступом. Использовать `fig.text()` или отдельные `axes` с фиксированными координатами в нормализованном пространстве (0–1), а НЕ через `tight_layout()` — он не справляется.
- Между рядами метрик: минимальный вертикальный зазор **40px** (в нормализованных координатах ~0.04).

**Конкретная схема позиционирования (нормализованные координаты):**

```python
# Размер фигуры

# y=0.93 — подзаголовок (дата, период)

# Позиция: x=0.35, y=0.22

# Состояние + Био-возраст — текстом справа от паутинки

# === НИЖНЯЯ ПАНЕЛЬ МЕТРИК ===
# Ряд 1: y=0.14 (значения), y=0.11 (названия), y=0.08 (цветные точки)
# Ряд 2: y=0.05 (значения), y=0.02 (названия)
```

б) **Общий балл "43 /100" залезает на BL-бар.**  
Сейчас общий балл размещён прямо под BL, и при низких значениях BL (0%) полоска исчезает, а текст балла визуально "висит в воздухе" или налезает.

**Как исправить:**
- Общий балл вынести в **отдельную зону** между прогресс-барами и нижней панелью метрик.
- Сделать его крупным (fontsize=36, fontweight='bold'), с подписью "/100" мельче (fontsize=16).
- Координаты: `fig.text(0.27, 0.22, '43', fontsize=36, fontweight='bold', color=score_color)` + `fig.text(0.35, 0.23, '/100', fontsize=14, color='gray')`.

**Как исправить:**
- Для подписей осей использовать фиксированную ширину поля: `ax.set_yticklabels(labels, fontsize=10)`, и выравнивать по правому краю с фиксированным отступом.

г) **Цветные индикаторы (зелёный/жёлтый/красный квадратик) в нижней панели:**

- Индикатор рисовать как `fig.text()` с символом ■ (или `matplotlib.patches.Rectangle`), с фиксированным `y` на 0.015 ниже значения.
- Или вообще убрать отдельный индикатор и вместо этого **красить само значение в нужный цвет** — это проще и чище.

**Пример:**

- Использовать `ax.text(..., wrap=True)` с ограничением bbox.

б) **Рамки блоков: слишком тонкие, плохо видны.**

**Как исправить:**
```python

for spine in ax.spines.values():
    spine.set_linewidth(2)
    spine.set_color(block_color)  # зелёный для S, оранжевый для W, бирюза для O, красный для R
```

в) **Вертикальное распределение текста внутри блока.**  
Сейчас тексты группируются вверху, а внизу блока — пустота.

**Как исправить:**
```python
def draw_swot_block(ax, title, items, title_color, border_color, bg_alpha=0.1):
    ax.set_facecolor('#1a1a2e')
    ax.set_xlim(0, 1)
    ax.set_ylim(0, 1)
    ax.axis('off')
    
    # Рамка
    for spine in ax.spines.values():
        spine.set_visible(True)
        spine.set_linewidth(2.5)
        spine.set_color(border_color)
    
    # Заголовок блока
    ax.text(0.5, 0.90, title, fontsize=14, fontweight='bold',
            color=title_color, ha='center', va='top')
    
    # Элементы — равномерно распределяем по высоте
    n = len(items)
    if n > 0:
        y_start = 0.78
        y_end = 0.08
        y_step = (y_start - y_end) / max(n, 1)
        
        for i, item in enumerate(items):
            y = y_start - i * y_step
            # Перенос длинных строк
            wrapped = textwrap.fill(item, width=48)
            ax.text(0.06, y, f'• {wrapped}', fontsize=10,
                    color='#e0e0e0', va='top',
                    fontfamily='sans-serif',
                    linespacing=1.4)
```

### 2.1 ПРОБЛЕМА

Сейчас SWOT использует шаблонные фразы вида:
```

"Стрессоустойчивость 70% — хорошие ресурсы для стандартных задач"

```

Эти фразы **одинаковые при каждом запуске** и **слишком короткие/шаблонные**. В каталоге `NEUROHRV_TEXT_CATALOG_FULL.md` прописаны **полные персонализированные формулировки** с вариативностью.

### 2.2 ЧТО ДОЛЖНО БЫТЬ

Каждая формулировка в SWOT должна:
1. Определяться **уровнем** метрики (critical / low / medium / good / excellent)
2. Выбираться из **3 вариантов** (ротация по `hash`)
3. Использовать **контекст** (🧠 когнитивный / 💪 физический / 🔄 универсальный)
4. Содержать **конкретное значение** метрики (`{value}`)

**Шаг 2: Функции определения уровня и выбора текста**

```python
import hashlib

def get_level(value_percent):
    """Определяет уровень по значению оси (0-100%)"""
    if value_percent >= 80:
        return 'excellent'
    elif value_percent >= 60:
        return 'good'
    elif value_percent >= 40:
        return 'medium'
    elif value_percent >= 16:
        return 'low'
    else:
        return 'critical'


def get_variant_index(user_id, date_str, metric_code):
    """
    Выбирает вариант текста (0, 1, или 2) на основе хэша.
    Один пользователь в один день видит один вариант.
    В другой день или при другом замере — другой вариант.
    """
    seed = f"{user_id}_{date_str}_{metric_code}"
    h = hashlib.md5(seed.encode()).hexdigest()
    return int(h, 16) % 3


def get_axis_text(axis, value_percent, context, user_id, date_str):
    """
    Возвращает полную персонализированную формулировку для оси.
    
    Args:
        axis: 'RD', 'SR', 'AD', 'FL', 'RC', 'EN', 'BL'
        value_percent: 0-100
        context: 'cognitive', 'sport', 'universal'
        user_id: ID пользователя (для вариативности)
        date_str: дата замера (для вариативности)
    
    Returns:
        str: Полная формулировка с подставленным значением
    """
    level = get_level(value_percent)
    variant = get_variant_index(user_id, date_str, axis)
    
    texts = AXIS_TEXTS[axis][level][context]
    template = texts[variant]
    
    return template.format(value=value_percent)


def get_swot_strengths(axes_scores, context, user_id, date_str):
    """Формирует блок Сильных сторон: оси с value >= 60%"""
    strengths = []
    for axis, value in sorted(axes_scores.items(), key=lambda x: -x[1]):
        if value >= 60:
            text = get_axis_text(axis, value, context, user_id, date_str)
            strengths.append(text)
    return strengths[:4]  # максимум 4


def get_swot_weaknesses(axes_scores, context, user_id, date_str):
    """Формирует блок Ограничений: оси с value < 40%"""
    weaknesses = []
    for axis, value in sorted(axes_scores.items(), key=lambda x: x[1]):
        if value < 40:
            text = get_axis_text(axis, value, context, user_id, date_str)
            weaknesses.append(text)
    return weaknesses[:4]


def get_swot_opportunities(axes_scores, user_id, date_str):
    """Формирует блок Возможностей: рекомендации для осей < 60%"""
    opps = []
    for axis, value in sorted(axes_scores.items(), key=lambda x: x[1]):
        if value < 60 and axis in OPPORTUNITIES:
            variant = get_variant_index(user_id, date_str, f'opp_{axis}')
            opps.append(OPPORTUNITIES[axis][variant])
    return opps[:3]  # максимум 3


def get_swot_risks(metrics, user_id, date_str):
    """Формирует блок Рисков: по критическим порогам метрик"""
    risks = []
    for i, risk in enumerate(RISKS):
        if risk['condition'](metrics):
            variant = get_variant_index(user_id, date_str, f'risk_{i}')
            template = risk['texts'][variant]
            try:
                text = template.format(**metrics)
            except (KeyError, ValueError):
                text = template  # если переменная не найдена — текст как есть
            risks.append(text)
    return risks[:4]  # максимум 4
```

## ЧАСТЬ 3: ОБРЕЗКА ТЕКСТОВ В SWOT

### 3.1 Проблема

Формулировки из каталога **длиннее** прежних шаблонов. Например:

**Было:** `"Готовность 51% — достаточные ресурсы"` (33 символа)  
**Стало:** `"Готовность 51% — ресурсы ограничены. Рекомендуется сосредоточиться на рутинных задачах и избегать важных решений."` (113 символов)

### 3.2 Решение: адаптивное усечение для SWOT

В SWOT-блоке ограниченное место. Нужна **короткая версия** для SWOT и **полная версия** для отдельного текстового отчёта (если будет).

**Правило усечения для SWOT-дашборда:**

```python
def truncate_for_swot(text, max_chars=85):
    """
    Усекает текст для SWOT-блока.
    Берёт первое предложение или обрезает по max_chars.
    """
    # Попробовать первое предложение
    sentences = text.split('. ')
    first = sentences[0]
    if not first.endswith('.'):
        first += '.'
    
    if len(first) <= max_chars:
        return first
    
    # Если и первое предложение длинное — обрезаем по словам
    words = first.split()
    result = ''
    for word in words:
        if len(result) + len(word) + 1 > max_chars - 3:
            break
        result += (' ' if result else '') + word
    return result + '...'
```

Таким образом в SWOT-блоке будет:
```
• Готовность 51% — ресурсы ограничены.
```
Вместо обрезанного:
```
• Готовность 51% — достаточные ресурсы  ← слишком обобщённо
```
И вместо:
```
• Готовность 51% — ресурсы ограничены. Рекомендуется 
  сосредоточиться на рутинных задачах и избегать важны  ← обрезано
```
