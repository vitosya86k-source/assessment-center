# 📱 Интеграция с Android приложением

## Простое подключение через файлы

**Вам НЕ нужно дорабатывать Android приложение!** Просто настройте сохранение CSV файлов.

## Как это работает

1. **Android приложение** сохраняет CSV файлы с данными HRV
2. **Синхронизация** файлов на сервер (через ADB, облако или локальную сеть)
3. **Бот** автоматически читает файлы из папки `android_sync/`

## Настройка Android приложения

### Вариант 1: Сохранение в общую папку

Настройте Android приложение сохранять CSV файлы в:
```
/storage/emulated/0/HRV_Monitor/data/
```

Или в любую другую папку, которую вы укажете.

### Вариант 2: Использование существующей папки

Если у вас уже есть Android приложение, которое сохраняет данные, просто укажите путь к этой папке в `config.py`:

```python
ANDROID_DATA_PATH = "/путь/к/вашей/папке"
```

## Синхронизация файлов

### Способ 1: ADB (Android Debug Bridge)

```bash
# Подключите Android устройство через USB
adb devices

# Синхронизация файлов
adb pull /storage/emulated/0/HRV_Monitor/data/*.csv ./android_sync/
```

### Способ 2: Облачное хранилище

1. Android приложение сохраняет в Google Drive / Dropbox
2. На сервере скачиваете файлы в `android_sync/`

### Способ 3: Локальная сеть (FTP/SMB)

Настройте Android приложение отправлять файлы на FTP/SMB сервер, откуда бот будет их забирать.

### Способ 4: Автоматическая синхронизация (скрипт)

Создайте скрипт `sync_android.sh`:

```bash
#!/bin/bash
# Синхронизация данных с Android устройства

ANDROID_PATH="/storage/emulated/0/HRV_Monitor/data"
LOCAL_PATH="./android_sync"

# Через ADB
adb pull "$ANDROID_PATH"/*.csv "$LOCAL_PATH/" 2>/dev/null

# Или через rsync по сети
# rsync -av user@android-device:/path/to/data/ "$LOCAL_PATH/"
```

Запускайте этот скрипт периодически (через cron) или вручную перед анализом.

## Использование в боте

После синхронизации файлов используйте команду `/analyze` в боте. Бот автоматически найдет последний CSV файл в папке `android_sync/`.

Или отправьте файл напрямую боту через Telegram.

## Формат CSV файла

Android приложение должно сохранять CSV с минимум колонкой:

```csv
Heart_Rate_bpm,Timestamp_ISO
75,2025-01-22 17:24:45
76,2025-01-22 17:24:46
...
```

Остальные колонки (RR_Interval_ms, Approximate_RMSSD и т.д.) опциональны - они будут рассчитаны автоматически.

## Проверка работы

1. Сохраните тестовый CSV файл в `android_sync/`
2. Запустите бота
3. Используйте `/analyze` - бот должен найти и обработать файл

## Автоматизация

Для автоматической синхронизации добавьте в cron:

```bash
# Каждые 5 минут синхронизировать данные
*/5 * * * * cd /path/to/bot && ./sync_android.sh
```

Или используйте systemd timer для более надежной работы.
