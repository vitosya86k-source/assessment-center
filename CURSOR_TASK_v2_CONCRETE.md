# ОШИБКА 1: ПОДПИСИ СДВИНУТЫ

## Симптом:
```
На экране:                    Должно быть:
RD                            RD
  Готовность    [████]          Готовность       [████]

SR                            SR
  Готовность    [████████]      Стрессоустойчивость [████████]
  ↑ ОШИБКА!

AD                            AD
  Стрессоустойчивость           Адаптивность
  ↑ ОШИБКА!
```

## Как найти в коде:

### Шаг 1: Найти где определяются названия
```python
# Искать что-то вроде:
axis_names = ['Готовность', 'Стрессоустойчивость', 'Адаптивность', ...]
# или
LABELS = {...}
# или
names = {...}
```

### Шаг 2: Найти цикл отрисовки
```python
# Искать что-то вроде:
for i, axis in enumerate(axes):
    # Здесь рисуются подписи
    ax.text(x, y, name, ...)
```

## Вероятная причина:
Подпись рисуется НЕ на той же итерации что и бар.

## Конкретное исправление:

### Вариант A: Если используется enumerate со сдвигом
```python
# БЫЛО (неправильно):
for i, axis in enumerate(axes):
    name = names[i+1]  # ← сдвиг!
    
# НАДО:
for i, axis in enumerate(axes):
    name = names[i]  # ← без сдвига
```

### Вариант B: Если подписи в отдельном цикле
```python
# БЫЛО (неправильно):
# Цикл 1: рисуем бары
for i, axis in enumerate(axes):
    draw_bar(axis, values[axis])

# Цикл 2: рисуем подписи (с другим порядком!)
for i, name in enumerate(names):
    draw_label(name, y_positions[i])

# НАДО: объединить в один цикл
for i, axis in enumerate(axes):
    value = values[axis]
    name = AXIS_NAMES[axis]  # получать по ключу, не по индексу
    y = start_y - i * row_height
    
    draw_bar(y, value)
    draw_label(y, name)
```

### Вариант C: Использовать словарь вместо списка
```python
# Определить словарь:
AXIS_NAMES = {
    'RD': 'Готовность',
    'SR': 'Стрессоустойчивость',
    'AD': 'Адаптивность',
    'FL': 'Гибкость НС',
    'RC': 'Восстановление',
    'EN': 'Выносливость',
    'BL': 'Баланс'
}

# В цикле отрисовки:
for axis in ['RD', 'SR', 'AD', 'FL', 'RC', 'EN', 'BL']:
    name = AXIS_NAMES[axis]  # всегда правильное соответствие
    value = scores[axis]
    # ... рисуем
```

# ОШИБКА 4: БИОЛОГИЧЕСКИЙ ВОЗРАСТ 74 ГОДА

## Данные с экрана:
- RMSSD = 8.9 мс
- SDNN = 36.7 мс
- Stress Index = 42.5

## Проверка формулы:
```python
rmssd = 8.9  # мс — очень низкий!

# По формуле:
# rmssd < 10: base_age = 75 + (10 - rmssd) * 2.0
base_age = 75 + (10 - 8.9) * 2.0 = 75 + 2.2 = 77.2

# Коррекция по SDNN (36.7 > 30, < 100): без изменений
# Коррекция по SI (42.5 < 100): base_age -= 3 → 74.2

# Итого: 74 года — ФОРМУЛА РАБОТАЕТ ПРАВИЛЬНО!
```

## Реальная проблема:
RMSSD = 8.9 мс — это КРИТИЧЕСКИ низкое значение!
Такое бывает при:
- Ошибке расчёта
- Неправильных входных данных
- Аритмии

## Что проверить:

### Проверка 1: Как рассчитывается RMSSD
```python
def calculate_rmssd(rr_intervals):
    """
    RMSSD = √(Σ(RRᵢ₊₁ - RRᵢ)² / (n-1))
    """
    # Проверить что rr_intervals в МИЛЛИСЕКУНДАХ
    print(f"DEBUG: RR range = {min(rr_intervals)} - {max(rr_intervals)} мс")
    # Должно быть ~600-1200 мс
    
    # Если значения ~0.6-1.2, то это СЕКУНДЫ — нужно умножить на 1000!
    
    differences = np.diff(rr_intervals)
    rmssd = np.sqrt(np.mean(differences ** 2))
    
    print(f"DEBUG: RMSSD = {rmssd} мс")
    return rmssd
```

### Проверка 2: Как рассчитываются RR интервалы из HR
```python
def hr_to_rr(hr_bpm):
    """
    RR (мс) = 60000 / HR (уд/мин)
    """
    # При HR = 60: RR = 1000 мс
    # При HR = 100: RR = 600 мс
    
    rr_ms = 60000 / hr_bpm  # результат в МИЛЛИСЕКУНДАХ
    return rr_ms
```

### Проверка 3: Откуда берутся данные
```python
# В коде найти где читаются данные:
hr_data = data['Heart_Rate_bpm']  # или другое название колонки

# Проверить что это действительно HR, а не что-то другое
print(f"DEBUG: HR range = {hr_data.min()} - {hr_data.max()}")
# Должно быть ~50-100 уд/мин
```

## Возможные причины низкого RMSSD:

### Причина A: RR в секундах вместо миллисекунд
```python
# БЫЛО:
rr = 60 / hr  # результат в секундах: 0.6-1.2

# НАДО:
rr = 60000 / hr  # результат в миллисекундах: 600-1200
```

### Причина B: Разницы RR слишком маленькие
При расчёте RR из HR с точностью до целых:
```
HR: 60, 61, 60, 61, 60...
RR: 1000, 984, 1000, 984, 1000...
diff: 16, 16, 16, 16...
RMSSD = 16 мс — низкий, но не 8.9
```

Если HR округляется сильнее или данные сглажены:
```
HR: 60, 60, 60, 60, 60... (почти константа)
RR: 1000, 1000, 1000...
diff: 0, 0, 0...
RMSSD ≈ 0 мс — критически низкий!
```

### Причина C: Используется другая переменная
```python
# Проверить что в биологический возраст передаётся именно RMSSD:
bio_age = calculate_biological_age(
    rmssd=metrics['rmssd'],  # ← проверить что это RMSSD, а не что-то другое
    sdnn=metrics['sdnn'],
    stress_index=metrics['stress_index']
)
```
