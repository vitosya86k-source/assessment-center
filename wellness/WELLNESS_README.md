## 1. Глаза — `eye_markers.py` ✅ ГОТОВО, проверено

Реальный видеоанализ через MediaPipe **FaceLandmarker (Tasks API)** + модель
`face_landmarker.task` (лежит рядом, 3.76 МБ, официальная MediaPipe).

`analyze(video_path) → dict`:
- `blink_rate_per_min` — частота морганий (по EAR);
- `perclos_pct` — доля времени с закрытыми глазами (утомление);
- `saccade_rate_per_min` — частота скачков взгляда;
- `fixation_stability` — устойчивость фиксации (0..1);
- `nystagmus_proxy` — ритмичные горизонт. колебания зрачка 2–8 Гц (0..1);
- `vestibular_signal` — «норма / неустойчивая фиксация / ритмичные колебания — обратить внимание».

**Проверено на реальном видео-кружке** (384×384, 448 кадров с лицом): моргания 32/мин,
PERCLOS 7%, саккады 32/мин, нистагм 0.04. На мелких плитках Гранатума лицо не
детектится (корректно отдаёт «лица не видно»). Покрывает и PERCLOS-утомление для Neiry.

**Рамка:** поведенческие маркеры, НЕ диагноз. Нистагм/вестибулярный сигнал — повод присмотреться.

### Как подключить в кружок (state_snapshot.build_snapshot)
В ветке `if has_video:` рядом с `pulse`/`face`:
```python
import eye_markers
try:
    eye = eye_markers.analyze(str(src))
except Exception as e:
    eye = {"available": False, "reason": f"eye_markers error: {e}"}
```
и добавить `"eye": eye,` в словарь снапшота. В else-ветке `eye = {"available": False, "reason": "audio note"}`.

## 3. Давление — `bp_estimate.py` ✅ РАБОЧАЯ v1 (обучаемая/калибруемая)

Рамка строго «оценка, НЕ измерение мм рт.ст.». `estimate(pulse) → {sbp, dbp, tendency, confidence}`,
где `pulse` — результат `rppg_video.analyze` (ЧСС, ВСР, дыхание уже считаются).

**Подход — стандартный cuffless BP**, не выдумка: персонально-калибруемая регрессия
по надёжным rPPG-признакам. Коэффициенты **обучаемые** (`bp_fit.json`), здесь начальные.
Физиологические guard'ы: шумные с короткого видео ВСР/дыхание (вне диапазона) отбрасываются,
ЧСС остаётся опорной. Калибровка: `calibrate(cuff_sbp, cuff_dbp, pulse)` — один замер
манжетой в покое сдвигает базовую точку под человека.

**Проверено на реальном кружке:** ЧСС 68, ВСР 252 (шум → отброшен) → **111/72** «в норме»;
после калибровки манжетой 122/80 → **117/77**. Есть флаг `ENABLED=False` — выключить, если в деле не зайдёт.

**Как подключить в кружок:** после `pulse = rppg_video.analyze(...)` →
```python
import bp_estimate
bp = bp_estimate.estimate(pulse)
```
и `"bp": bp,` в снапшот.

**Upgrade-пути (реальные репо, вендорены в wellness/):**
- `PPG2ABP/` (`nibtehaz/PPG2ABP`) — одно-PPG → ABP-волна → SBP/DBP, DL-модель. Нужен TF/keras
  + скачать веса (отдельно). Точнее, но тяжелее. Подключить, когда заведём TF.
- `phase_shifted_rppg/` (`gyutaehwang/...`) — face+hand PPG, обучаемая, без весов (нужна тренировка V4V/MIMIC).
- Коммерческий SDK Binah.ai — vitals+BP spot-check по смартфону (проверять regulatory).
